整个应用分为三块：左侧仓库列表、中间代码区、底部提交/推送按钮。很直观。

1️⃣ 登录 GitHub 账号

1 双击桌面上的 GitHub Desktop 图标打开
首次启动界面
🔑
欢迎使用 GitHub Desktop
Sign in to GitHub.com Sign in to GitHub Enterprise
点击左侧蓝色按钮，登录 GitHub.com 账号
2 点击 "Sign in to GitHub.com"
浏览器授权页面
🌐
会自动弹出浏览器
浏览器中点击 "Authorize desktop" 授权
3 授权成功后回到桌面端，就登录好了

右上角会显示你的 GitHub 头像和用户名

如果浏览器没自动弹出，可以手动打开浏览器再试一次。

2️⃣ 配置用户名和邮箱（只用设置一次）

1 点击顶部菜单栏 File → Options
Options 设置窗口
⬤ Accounts ⬤ Integrations ⬤ Git ⬤ Appearance ⬤ Advanced
Name（用户名）
你的 GitHub 用户名
Email（邮箱）
你的 GitHub 注册邮箱
2 填写你的 GitHub 用户名和邮箱

Name：填写你的 GitHub 用户名

Email：填写注册 GitHub 时用的邮箱

点击 Save 保存

这一步只用做一次，以后提交代码都会使用这个信息。

仓库名只能用 字母、数字、-、_

勾选 README 选项，会自动生成一个说明文件

创建成功后的界面

发布仓库窗口

发布成功后，仓库就会出现在 github.com/你的用户名/仓库名

私有 / 公开： 勾选 "Keep this code private" 就是私有仓库，只有你能看到。不勾选就是公开仓库，所有人都能看到。

1 点击左上角 File → Clone repository
克隆仓库窗口
GitHub.com URL
输入仓库地址或搜索
──────────────────────────────
📁 你的用户名/repo名
📁 其他用户的公开仓库
2 选择要克隆的仓库

如果是你自己的仓库，列表里会直接显示，点击选中即可

选择保存路径

选择保存到电脑的哪个文件夹
D:\我的项目

3 点击 Clone 下载到本地

下载完成后会自动显示在 GitHub Desktop 中，可以开始编辑了

从 URL 克隆别人的仓库： 点击顶部的 URL 标签，粘贴仓库地址 https://github.com/用户名/仓库名.git

5️⃣ 修改代码并提交（Commit）

1 在仓库文件夹中添加或修改文件

打开仓库文件夹（右键仓库 → Show in File Explorer）

用记事本或 VS Code 编辑文件，保存

主界面 — 已检测到修改
Changes History
📄 M README.md （已修改）
📄 + index.html （新增）
📝 输入提交说明（必填）
Commit to main
也可以添加详细说明（可选）
2 填写提交说明

第一行：必填，简要说明改了啥（如"添加了首页"）

第二行：可选，详细描述修改内容

3 点击 Commit to main 完成提交

提交后修改就保存到本地仓库的历史中了

提交说明要写清楚，方便以后回顾。好的例子：修复了登录页面的按钮样式 vs 不好的例子：修改

6️⃣ 推送到 GitHub（Push）

1 提交完成后，顶部会出现推送按钮
等待推送
已提交 1 个更改
尚未推送到 GitHub
🔼 Push origin
2 点击 Push origin 上传到 GitHub

推送完成后，刷新 GitHub 网页就能看到更新

什么时候推送？ 每次 commit 后都可以 push 一次。
也可以连续 commit 多次，最后一起 push。

7️⃣ 从 GitHub 拉取更新（Pull）

1 点击顶部的 Fetch origin
拉取更新
⬇️
Fetch origin
如果有新内容，按钮会变成 Pull origin
2 如果有更新，按钮变成 Pull origin，点击下载

多人协作时，每次开始工作前先 Pull 一下，保持代码最新

❓ 常见问题

Q 推送时提示没有权限怎么办？

可能登录的账号不是仓库所有者。退出重新登录正确的账号：

File → Options → Accounts → Sign out → 重新登录

Q 提交错了想撤回怎么办？

在 History 中右键上次提交 → Undo commit（撤销提交）

Q 如何删除本地仓库？

右键仓库 → Remove（只是从 GitHub Desktop 中移除，不会删除硬盘文件）

彻底删除：在文件资源管理器中手动删除文件夹
